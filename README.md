# ZhoneMap 🗺️
> _"I hate my ISP's shitty security."_

**ZhoneMap** is a Bash and expect script designed to automate the extraction of Routing and ARP tables from legacy Zhone/DZS routers. It bypasses modern OpenSSH client restrictions to connect to older firmware, automates privilege escalation, and parses the raw CLI output into a clean, readable terminal format or a structured JSON file.

## ✨ Features
* **Legacy SSH Bypass**: Automatically forces `3des-cbc` and `ssh-rsa` ciphers so modern Linux distros don't reject the connection to the old router.

* **Pagination Automation**: Uses expect to automatically hit the spacebar on --More-- and Press any key to continue prompts.
 
* **Privilege Escalation**: Automatically drops into enable mode (# prompt) if required.
Dependency-Free JSON Export: Manually constructs JSON output so you don't need jq installed on your scanning machine.

* **Clean CLI**: Formats messy router output into a clean, colorized terminal view identifying Default Gateways, Loopbacks, Multicast routes, and active ARP clients.

## 📦 Prerequisites
You need a standard Linux environment with the following packages installed:
* `bash`
* `expect` (Crucial for the SSH automation)
* `openssh-client`
* `curl` (Only used to fetch your external IP address)

#### On Debian/Ubuntu:
```
sudo apt update
sudo apt install expect ssh curl
```

## 🚀 Usage
### Make the script executable:
```bash
chmod +x zhonemap.sh
```

### Basic Run (Interactive Password Prompt):
```bash
./zhonemap.sh -u admin -ip 192.168.1.1
```
### or simply
```
./zhonemap.sh
```

### Export to JSON:
```
./zhonemap.sh -o network_dump.json
```

### Debug Mode (If the router isn't responding as expected):
```
# Level 1: Shows script injection steps
./zhonemap.sh -v 

# Level 2: Pipes live SSH output to the screen
./zhonemap.sh -vv
```

### Minimal Mode (No colors/emojis, _good for logging_):
```
./zhonemap.sh -m
```

## ⚠️ Security Notes

* **External IP Check**: By default, this script pings api.ipify.org (and fallbacks) to grab your public IP for the report. If you are running this over a VPN or Tor and care about OPSEC, be aware that it makes an external HTTP request. If you don't like it just _REMOVE IT_.

* **Weak Ciphers**: This script intentionally uses weak SSH ciphers (`3des-cbc`). This is a mechanical necessity to communicate with the router, but it means the local network traffic is technically vulnerable to decryption if intercepted.
